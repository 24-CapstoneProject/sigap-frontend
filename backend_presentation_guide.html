1. Konsep Dasar & Teknologi Stack

Backend SIGAP menggunakan model RESTful API yang memisahkan logic server dengan tampilan antarmuka (frontend). Komunikasi data menggunakan format JSON melalui protokol HTTP.

Teknologi Komponen Fungsi Utama
Node.js Runtime Lingkungan pengeksekusi kode JavaScript di sisi server.
Express.js Web Framework Mengatur routing API, middleware, dan request/response lifecycle.
MySQL (mysql2) Database Menyimpan data relasional dengan koneksi cepat berbasis Promise.
JWT (JsonWebToken) Authentication Pengaman akses fitur berupa token terenkripsi.
Bcryptjs Password Hashing Algoritma searah untuk mengamankan password di database.
Multer File Upload Mengunggah gambar bukti peminjaman dan barang lost/found.
Xlsx (SheetJS) Excel Engine Membaca berkas import jadwal dan memproduksi laporan excel.

2. Struktur Direktori Proyek (Folder Structure)

Berikut adalah struktur direktori yang digunakan di bagian backend. Anda bisa menampilkan ini di VSCode untuk demo:

backend/
├── config/ # Konfigurasi koneksi (Database connection pool)
├── controllers/ # Logika bisnis utama (Pemrosesan data & query database)
├── database/ # Cetak biru database (schema.sql & seed.sql)
├── middleware/ # Filter keamanan (Auth token, Upload, Error handler)
├── routes/ # Jalur URL API (Pemetaan endpoint & validator input)
├── uploads/ # Direktori fisik penyimpanan file gambar/foto (.jpg, .png)
├── utils/ # Fungsi pembantu (Token generator, Password hashing)
└── server.js # Berkas utama (Entry point aplikasi)

3. Penjelasan Alur Kerja Utama (Core Workflows)

A. Proses Keamanan Login (Auth Workflow)

  • Registrasi: Password yang dimasukkan di-hash menggunakan algoritma bcrypt dengan 10 salt rounds sebelum disimpan ke database agar aman.
  • Login: Setelah verifikasi password berhasil, server membuat token JWT unik berisi data profil pengguna.
  • Otorisasi: Token ini dikirimkan kembali ke klien dan harus dikirim pada setiap request API terproteksi sebagai pembuktian akses yang sah.

B. Alur Integrasi Peminjaman Ruang & Proyektor

  • Saat pengajuan disetujui (Approved) oleh Admin dan mahasiswa mencentang butuh proyektor, sistem secara otomatis akan mengurangi stok proyektor di database (-1).
  • Entri peminjaman proyektor otomatis dibuat di log inventory_loans.
  • Ketika dibatalkan atau selesai, stok proyektor otomatis bertambah kembali (+1).

C. Ekspor & Impor Excel

  • Impor: Backend mengurai jadwal perkuliahan dari format Excel Untad, mendeteksi bentrok hari/jam, dan melakukan ekspansi tanggal otomatis per minggu sepanjang semester.
  • Ekspor: Menyajikan 3 lembar kerja dalam satu file Excel (Riwayat Peminjaman, Kondisi Barang, dan Data Barang) langsung diproses secara cepat dari memori RAM server.

4. Mekanisme Keamanan (Security Features)

🛡️ SQL Injection Prevention

Menggunakan Prepared Statements (?) pada driver mysql2 sehingga seluruh query disaring terlebih dahulu sebelum dieksekusi.

🔑 Irreversible Password Hashing

Password di-enkripsi secara searah menggunakan bcrypt, sehingga password asli tidak bisa dibaca walaupun database bocor.

🔒 Role-Based Access Control

Jalur API admin diproteksi ketat menggunakan middleware authorizeRole(['admin']) sehingga mahasiswa tidak bisa mengakses data rahasia.

5. Prediksi Tanya-Jawab Sidang Capstone

Pertanyaan 1

"Mengapa Anda memisahkan Frontend dan Backend secara terpisah?"

Jawaban: Pemisahan ini menggunakan arsitektur Decoupled (Decoupled Architecture). Keuntungannya adalah jika frontend bermasalah, backend tetap berjalan mandiri. Beban kerja server juga lebih ringan karena proses rendering tampilan terjadi di sisi browser klien (Client-Side Rendering), bukan di server.

Pertanyaan 2

"Apa itu Connection Pooling pada database MySQL Anda?"

Jawaban: Connection Pool adalah optimasi memori di mana server menyiapkan 10 koneksi database aktif yang terus menyala (connectionLimit: 10). Ketika ada request masuk, server tinggal meminjam koneksi yang ada alih-alih membuat koneksi TCP baru dari awal yang memakan waktu dan kinerja CPU.

Pertanyaan 3

"Bagaimana Anda menangani file sampah jika proses database gagal di tengah jalan?"

Jawaban: Di dalam controller, proses simpan database dibungkus dengan blok try-catch. Jika database gagal (misal bentrok jadwal), blok catch akan memanggil helper cleanupFile() untuk menghapus file gambar bukti yang baru terunggah menggunakan fs.unlinkSync(), sehingga disk penyimpanan server tetap bersih.
